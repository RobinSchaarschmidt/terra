# Terra Economics — Token Model

## Scope

This document defines the Terra token system as a framework for:
- representing rights and responsibilities  
- accounting for impact on planetary commons  
- coordinating economic activity across systems  

Tokens are not assets by default.  
They are instruments of measurement, allocation, and accountability.

---

## Core Principle

Value is not defined by market demand alone.

Value emerges from:
- contribution to system stability  
- responsible use of shared resources  
- long-term impact across time  

---

## Token Architecture

Terra defines four primary token classes:

- TERRA  
- GOVT  
- CREDIT  
- WORK  

Each token serves a distinct function.

---

## TERRA (Long-Term Governance Layer)

Purpose:
Represents long-term alignment with planetary stability.

Properties:
- limited issuance  
- non-inflationary by default  
- tied to proven long-term contribution  

Function:
- access to high-level governance processes  
- participation in system-wide decisions  
- weighting in long-term validation  

Constraint:
TERRA cannot be acquired purely through purchase.  
It must be earned through verified contribution over time.

---

## GOVT (Operational Governance)

Purpose:
Enables participation in ongoing decision processes.

Properties:
- dynamic supply  
- tied to active engagement  
- time-sensitive validity  

Function:
- voting on proposals  
- local and regional governance  
- signaling system preference  

Constraint:
GOVT weight is adjusted by trust and validation history.

---

## CREDIT (Value Representation)

Purpose:
Represents claim on produced value or system contribution.

Properties:
- issued based on measurable output  
- redeemable within Terra economy  
- linked to real activity  

Function:
- compensation  
- exchange medium  
- value accounting  

Constraint:
CREDIT issuance must be backed by verifiable contribution or resource flow.

---

## WORK (Activity Layer)

Purpose:
Captures real-world activity and contribution.

Properties:
- high-frequency issuance  
- local and task-specific  
- directly linked to actions  

Function:
- input signal for CREDIT generation  
- measurement of participation  
- micro-level economic tracking  

Constraint:
WORK must be validated to prevent false reporting.

---

## Value Function

All tokens interact through a dynamic value model:

V_task = F(S, D, Q, U, R, T)

Where:
- S = Scarcity  
- D = Distribution  
- Q = Quality  
- U = Utility  
- R = Risk / Harm  
- T = Time impact  

This function determines:
- token issuance  
- weighting  
- system response  

---

## Resource Flow Mapping

Every economic action must define:

- resource input  
- transformation process  
- output value  
- system impact  

Unmapped flows are invalid.

---

## Impact Accounting

All activity is evaluated against:

- planetary commons usage  
- regeneration capacity  
- externalized cost  

Negative externalities are converted into:
- reduced issuance  
- increased obligation  
- system penalties  

---

## Time Weighting

Value is adjusted based on temporal impact:

- short-term gain is discounted  
- long-term stability is amplified  

Delayed harm increases liability.

---

## Issuance Logic

Token creation follows:

- verified activity → WORK  
- validated contribution → CREDIT  
- sustained impact → TERRA  

GOVT emerges from active participation.

---

## Anti-Speculation Constraint

Tokens are not designed for passive accumulation.

Mechanisms include:
- decay functions  
- usage requirements  
- contribution thresholds  

Holding without contribution reduces influence.

---

## Trust Integration

All token weights are influenced by:

- validation history  
- consistency of contribution  
- cross-system verification  

Trust is dynamic and continuously updated.

---

## System Objective

The Terra economic system ensures:

- alignment between value and reality  
- accountability for resource use  
- fair distribution of benefit and responsibility  

---

End of Token Model
